# Contributing to FiroBlocks UI

Thanks for your interest in contributing. This is a community infrastructure project maintained by [NexusOcean](https://nexusocean.io).

## Expectations

- Open an issue before starting work on a PR — this avoids duplicated effort
- PRs are reviewed on a best-effort basis
- Keep changes focused — one concern per PR
- This project follows [Semantic Versioning](https://semver.org)
- Unit tests should be written as needed and current tests must pass when using `pnpm test`

## Development Setup

See [README.md](./README.md) for environment setup. You will need a running FiroBlocks API and Sync instance.

## Guidelines

- TypeScript is strictly enforced
- Follow existing component and directory structure
- State management and data fetching should follow established patterns
- Do not commit `.env` or any configuration secrets

## Submitting a PR

1. Fork the repo and create a branch from `main`
2. Make your changes with clear, descriptive commits
3. Open a PR with a summary of what changed and why
